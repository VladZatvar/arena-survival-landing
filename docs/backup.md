# Інструкція з резервного копіювання

## 1. Мета резервного копіювання

Резервне копіювання потрібне для збереження:

- вихідного коду
- документації
- конфігураційних файлів
- допоміжних скриптів

## 2. Що саме потрібно зберігати

Для цього проєкту важливо створювати копії:

- `index.html`
- `style.css`
- `script.js`
- `README.md`
- папки `docs`
- файлу `package.json`
- файлу `package-lock.json`

## 3. Спосіб резервного копіювання

Основний спосіб резервного копіювання:

- збереження всіх змін у Git-репозиторії
- надсилання змін на GitHub

Додатковий спосіб:

- копіювання проєкту в окрему папку з датою створення резервної копії

## 4. Частота створення копій

Рекомендується виконувати резервне копіювання:

- перед великими змінами
- перед оновленням проєкту
- перед видаленням або заміною файлів
- після завершення важливого етапу роботи

## 5. Перевірка резервної копії

Потрібно перевіряти:

- чи всі важливі файли потрапили в копію
- чи відкривається резервна папка
- чи можна відновити проєкт із копії

## 6. Відновлення

Для відновлення можна:

- клонувати репозиторій із GitHub
- або повернути файли з локальної резервної копії

Основна команда для відновлення з GitHub:

```bash
git clone https://github.com/VladZatvar/arena-survival-landing.git
```
