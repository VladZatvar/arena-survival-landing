# Документація системи логування та обробки помилок

## 1. Загальна інформація

У проєкті `arena-survival-landing` реалізовано базову систему логування для клієнтської частини вебзастосунку.

Система логування використовується для:

- відстеження запуску сторінки;
- фіксації дій користувача;
- логування попереджень;
- логування помилок;
- збереження контекстної інформації про події.

## 2. Реалізація

Для логування створено окремий файл `logger.js`, який містить:

- рівні логування `DEBUG`, `INFO`, `WARN`, `ERROR`;
- функції для виведення повідомлень у консоль;
- перевірку поточного рівня логування;
- зміну рівня логування без зміни коду програми.

## 3. Рівні логування

У проєкті використовуються такі рівні логування:

- `DEBUG` — детальна технічна інформація;
- `INFO` — основні події нормальної роботи;
- `WARN` — попередження про нетипові ситуації;
- `ERROR` — помилки, що впливають на коректність роботи.

## 4. Зміна рівня логування

Поточний рівень логування зберігається у `localStorage` за ключем:

arena_survival_log_level

За замовчуванням використовується рівень `INFO`.

Змінити рівень логування можна через консоль браузера:

window.appLogger.setLogLevel("DEBUG");

Приклад:

- `DEBUG` — показує всі повідомлення;
- `INFO` — показує INFO, WARN, ERROR;
- `WARN` — показує WARN, ERROR;
- `ERROR` — показує тільки ERROR.

## 5. Формат повідомлень

Формат логів включає:

- час події;
- рівень логування;
- текст повідомлення;
- контекстну інформацію.

Приклад повідомлення:

[2026-03-27T12:00:00.000Z] [INFO] Ініціалізація лендінгу розпочата | context={"page":"/index.html"}

## 6. Обробка помилок

У файлі `script.js` реалізовано:

- `try/catch` для критичних дій;
- створення структурованого опису помилки;
- унікальні коди помилок;
- показ інформативного повідомлення користувачу;
- глобальне логування JavaScript-помилок через `window.addEventListener("error", ...)`.

## 7. Приклади кодів помилок

У проєкті використовуються такі приклади кодів:

- `NAV-001` — не знайдено цільову секцію;
- `NAV-500` — помилка під час прокрутки;
- `UI-001` — не знайдено елемент футера;
- `UI-002` — не знайдено кнопку повернення догори;
- `APP-500` — критична помилка ініціалізації сторінки;
- `JS-500` — глобальна JavaScript-помилка.

## 8. Повідомлення для користувача

Для користувача реалізовано прості й зрозумілі повідомлення без зайвих технічних деталей.
У разі помилки на сторінці зверху відображається повідомлення про проблему та рекомендація спробувати ще раз.

Також створено кастомну сторінку `404.html` для випадків переходу на неіснуючу адресу.

## 9. Обмеження реалізації

У поточній реалізації логування виконується через консоль браузера.
Запис логів у файл, ротація логів і централізований збір логів не реалізовані, оскільки проєкт є статичним вебсайтом без серверної частини.

## 10. Висновок

Реалізована система логування дозволяє відстежувати основні події роботи сторінки, виявляти типові проблеми та швидше діагностувати помилки під час розробки й тестування.
