# Налаштування linting у проєкті arena-survival-landing

## 1. Обрані інструменти

Для перевірки якості коду в проєкті були обрані такі інструменти:

- **HTMLHint** — для перевірки HTML-розмітки
- **Stylelint** — для перевірки CSS-стилів
- **Prettier** — для автоматичного форматування файлів

## 2. Причини вибору

Було обрано саме ці інструменти, оскільки проєкт є статичним вебсайтом і складається переважно з HTML та CSS.

- **HTMLHint** дозволяє виявляти помилки та неточності в HTML
- **Stylelint** допомагає підтримувати єдиний стиль CSS-коду
- **Prettier** автоматично форматує код і покращує його читабельність

## 3. Налаштовані правила

### HTMLHint

Для HTML були налаштовані базові правила:

- використання малих літер у назвах тегів
- використання малих літер у назвах атрибутів
- використання подвійних лапок в атрибутах
- наявність `DOCTYPE` на початку документа
- перевірка парності тегів
- унікальність `id`
- заборона порожніх `src`
- наявність тега `title`

### Stylelint

Для CSS використано стандартну конфігурацію `stylelint-config-standard`, а також додатково налаштовано:

- використання повної форми кольору
- вимкнено перевірку шаблону назв класів

### Prettier

Для форматування встановлено:

- відступ 2 пробіли
- без використання табуляції
- подвійні лапки

## 4. Ігнорування файлів

Щоб зайві файли не перевірялися, були додані файли ігнорування:

- `.stylelintignore`
- `.prettierignore`

У них виключено:

- `node_modules/`
- `package-lock.json`

## 5. Команди для запуску

У файлі `package.json` додано такі команди:

- `npm run lint:html` — перевірка HTML
- `npm run lint:css` — перевірка CSS
- `npm run format:check` — перевірка форматування
- `npm run format` — автоматичне форматування
- `npm run check` — комплексна перевірка всього проєкту

## 6. Виправлені проблеми

Під час запуску лінтера було знайдено проблеми у файлі `style.css`, зокрема:

- застарілий запис кольору через `rgba`
- не рекомендований запис альфа-значення
- застарілий запис media query

Після внесення змін усі знайдені проблеми були виправлені.

Також було виконано автоматичне форматування файлів:

- `index.html`
- `style.css`
- `README.md`

## 7. Pre-commit hook

Для автоматичної перевірки коду перед створенням коміту було налаштовано **Husky**.

У файлі `.husky/pre-commit` додано запуск команди:

```sh
npm run check
```
